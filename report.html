排序算法心得報告

本頁面整理五種排序法的基本原理、複雜度分析，並透過程式模擬比較不同演算法在不同資料規模下的效能差異。

1. 五個排序法介紹

以下介紹五種排序法的基本原理與操作方式：

  • 氣泡排序（Bubble Sort）：反覆比較相鄰兩個元素，如果順序錯誤就交換。每次遍歷後，最大的元素會移動到序列尾端。
  • 選擇排序（Selection Sort）：每輪選出剩餘元素中的最小值，將其放到已排序區域的末端。適合概念理解，但交換次數較少。
  • 插入排序（Insertion Sort）：逐一將元素插入到前面已排序區域的正確位置。對於近乎已排序的資料特別有效。
  • 合併排序（Merge Sort）：使用分治策略，先遞迴拆分陣列為較小區段，再合併已排序的子陣列，產生完整排序結果。
  • 快速排序（Quick Sort）：選擇一個樞紐元素（pivot），將資料分成小於與大於樞紐的兩個子陣列，遞迴排序後拼接結果。

2. 複雜度分析

比較這五種排序法的時間與空間複雜度：

排序法 時間複雜度 空間複雜度 穩定性
氣泡排序 最佳 O(n)、平均 O(n²)、最差 O(n²) O(1) 穩定
選擇排序 最佳 O(n²)、平均 O(n²)、最差 O(n²) O(1) 不穩定
插入排序 最佳 O(n)、平均 O(n²)、最差 O(n²) O(1) 穩定
合併排序 最佳 O(n log n)、平均 O(n log n)、最差 O(n log n) O(n) 穩定
快速排序 最佳 O(n log n)、平均 O(n log n)、最差 O(n²) O(log n)（遞迴堆疊） 不穩定

綜合分析可知，當資料量較小或接近已排序時，氣泡排序和插入排序表現還算合理；而對大資料量，合併排序與快速排序通常更高效。
